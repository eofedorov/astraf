# HR GPS Logger

Простое Android-приложение на Kotlin и Jetpack Compose:

- подключение к BLE-пульсометру со стандартным Heart Rate Service;
- отображение пульса (BPM) в реальном времени;
- получение GPS-координат через Fused Location Provider;
- запись данных в локальный CSV-файл;
- фоновая запись при свёрнутом приложении и выключенном экране (foreground service + уведомление);
- главный экран поверх блокировки и автозапуск при включении телефона (без ввода PIN/пароля на поддерживаемых устройствах).

## Сборка

1. Установите Android Studio с Android SDK 35.
2. Скопируйте `local.properties.example` в `local.properties` и укажите `sdk.dir`.
3. Откройте папку проекта `astraf` в Android Studio.
4. Соберите APK: **Build → Build Bundle(s) / APK(s) → Build APK(s)**  
   или из терминала:

```bash
gradlew.bat assembleDebug
```

> Если путь к проекту содержит кириллицу (например, имя пользователя Windows), в `gradle.properties` уже включено `android.overridePathCheck=true`.

## Ручная проверка на устройстве

1. Установите debug APK на реальный телефон с Android 8+.
2. После первой установки при необходимости разрешите приложению показываться поверх экрана блокировки в настройках системы (зависит от производителя).
3. Включите Bluetooth и геолокацию.
4. Запустите приложение и нажмите **Выдать разрешения**.
5. Наденьте/включите пульсометр в режим трансляции BLE.
6. Нажмите **Scan**, выберите устройство в списке, нажмите **Connect**.
7. Дождитесь появления BPM и GPS-координат на экране.
8. Нажмите **Start logging** — появится постоянное уведомление; запись продолжится при выключенном экране.
9. При необходимости нажмите **Отключить оптимизацию батареи**, чтобы система не останавливала GPS/BLE.
10. Сверните приложение или выключите экран, подождите 30–60 секунд.
11. Остановите запись кнопкой **Stop logging** или действием **Остановить** в уведомлении.
12. Проверьте CSV через Android Studio **Device Explorer**:
   - путь: `/data/data/com.astraf.hrgpslogger/files/`
   - файл: `hr_gps_<timestamp>.csv`
   - заголовок: `timestamp,bpm,latitude,longitude,accuracy_m`

## Ограничения

- Работает только с пульсометрами, поддерживающими BLE Heart Rate Service (`0x180D`).
- При случайном завершении процесса запись восстанавливается автоматически (сохранённое состояние + перезапуск foreground service).
- После перезагрузки телефона запись тоже возобновится, если была активна.
- Явная остановка — только кнопкой **Stop logging** или **Остановить** в уведомлении.
- На Android 13+ нужно разрешение на уведомления для фоновой записи.
